The All-Knowing Mind

AKM is a multi-language FPS game server browser. AKM is written in Java, making it operating system agnostic. It should run on any platform that has a JVM available and is supported by the SWT GUI library. There are multiple install packages because the SWT library is specific to the OS and JVM.

What's New in version 0.2?

This version includes most of the functionality that was missing from the previous version. It also has an installer to make installation a breeze. The GeoIP database has been updated. New features include the following:

  • Game autodetect
  • Game configuration dialog
  • Game engine configuration
  • Master server configuration
  • Per-game customizable folders
  • Add arbitrary servers by IP address
  • Drag and drop servers into folders
  • User preferences dialog

Supported Games

AKM currently supports:

  • Urban Terror
  • Quake 3
  • Wolfenstein: Enemy Territory
  • Enemy Territory: Quake Wars
  • Return to Castle Wolfenstein
  • Nexuiz
  • OpenArena
  • World of Padman
  • Warsow

The Half Life games and their derivatives, the Battlefield series, and the CoD games will eventually be supported. Any games that use id's Quake 3 protocols should be trivial to add without any code changes. If you want support for additional games, please submit a feature request via the project website.

Installation

A JRE version 1.5 or later must be installed. In Windows, you should be able to just double click on the jar file that you downloaded. You may get a warning popup indicating you need a Java development environment, but you can ignore this. It will still install.

To start installation in Linux, open a terminal window, cd to the directory containing the installation jar file, and type:

java -jar akm-install-<version>.jar

Installation is fairly self-explanatory. The biggest decision you need to make is where to install AKM. You can also choose to install shortcuts on your desktop and/or in the start menu. In the menu, it will be installed in "Games" by default. It is recommended that you install the shortcuts and use them to launch AKM rather than creating your own.

*************IMPORTANT LINUX NOTE***************

After you have completed the installation, you must give the akm.jar file execute permission. For some reason, the installer doesn't do it, though it's supposed to. To set the permissions, Open a terminal window, cd to the AKM installation directory (~/akm by default), and type:

chmod +x akm.jar

You will then be able to launch AKM with the shortcuts.

Uninstallation

An uninstall jar file is created by default in the Uninstall subdirectory of your installation directory. To completely uninstall AKM, open a terminal window, cd to the Uninstall directory, and type the command:

java -jar uninstall.jar

Game Configuration

The first time you run AKM, it will attempt to find the supported games in their default locations. On Windows platforms, those will be whatever the default installation directory is for the particular game. On Linux platforms, AKM will look in /usr/local. Any games found will automatically be enabled, as will all of their mods. If no games are found in the default locations, then the game configuration dialog will appear allowing you to choose which games are installed and where they are located. For a game to appear in the tree on the left side of the display, 'Show Game' must be checked. You can display games even if they are not installed. If you want to be able to launch a game from AKM, then it must be installed at the location indicated, and 'Game Installed' must be checked. You can also decide which mods you want to display.

You should not modify any of the game engine or master server settings unless you have a very specific reason for doing so. However, if you play Quake 3, you may want to restrict the servers to protocol 68. All protocol versions are displayed by default, but you will need a version switcher program, such as this one (for Windows only) to be able to play all versions.

Recovery

If you make any changes that make it impossible to scan for servers or launch a particular game, you can delete usergames.xml and AKM will rebuild it to the default values. usergames.xml is located in the data subdirectory under the installation directory. Do not modify or delete any other files in that directory or you may render AKM unusable.

Notes on Urban Terror 4

Urban Terror 4 is configured as a standalone game. This default configuration will launch the ioUrbanTerror client if the server is an ioq3 server and it will launch the quake3 client if the server is a Q3 server. You can change this behavior if you only have one of the two clients installed by removing the game engine(s) entries that you don't have installed using the game configuration dialog. For instance, if you don't have Quake 3 installed, remove all entries except for the ioq3 entry. If you don't have ioq3 installed, remove all of the entries that use it and only leave the q3 entry.

System Requirements

Windows XP or Linux (Mac OS X to be supported in a future release). It very well may work on other versions of Windows, but it hasn't been tested.

Java 1.5 Standard Edition is required at a minimum.

Directory Structure

Linux:

/ - Java jar files

data/ - GeoIP database and the XML configuration files

icons/ - all icons

flags/

game/

misc/

toolbar/

images/ - all non-icon images (maps, application, etc.)

maps/

<game1>

<game2>

<...>

resources/ - language and resource files

The log file is in ~/.akm.

Windows:

The default installation directory is C:\Program Files\The All-Knowing Mind\. The base directory has the following structure:

\ - Java jar files

data\ - GeoIP database (GeoIP.dat)

icons\ - all icons

flags\

game\

misc\

toolbar\

images\ - all non-icon images (maps, application, etc.)

maps\

<game1>

<game2>

<...>

resources\ - configuration and language files

The log file is in C:\Documents and Settings\<username>\Application Data\Dilireum\The All-Knowing Mind.

If you want to create your own Desktop shortcut from scratch, the easiest way is to create a new shortcut to javaw.exe. Then select the the shortcut properties and append "-jar org.dilireum.akm.jar" to the Target, and change the "Start in:" field to "C:\Program Files\The All-Knowing Mind". Alternatively, you can simply create a shortcut to "C:\Program Files\The All-Knowing Mind\akm.bat", but you will have to put up with an annoying extra DOS window if you do.

Map Images

You can add additional map images as you like. All map files must be jpeg format. They will automatically be compressed to 200x200. The file name must match the map name as reported by servers. You should make use all lower case since AKM will translate all map names reported by servers to lowercase before attempting to find a matching file name.

Reporting Bugs

AKM is now fully functional, but it is still in the pre-alpha development phase and there will be bugs. If you find any, please report them here:

http://sourceforge.net/tracker2/?func=add&group_id=243818&atid=1123681.

Provide as much detailed information as possible to help us track it down. There is a log file for AKM in ~/.akm/log on Linux platforms and in C:\Documents and Settings\<username>\Application Data\Dilireum\The All-Knowing Mind\log on Windows machines. Any stack trace from the log file plus a description of what you were doing when it crashed would be helpful.

If you would like to contribute, I'm looking for talent, whether it be doing language translations, testing, creating map images, web site design, graphics, researching game protocols, or hard core hacking.
